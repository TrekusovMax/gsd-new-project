### 2. Drop Zone

**States and their exact visual appearance:**

#### State: idle (default)
- Border: 2px dashed `#303050`, `border-radius: 8px`
- Background: `colorBgBase` (`#0d0d0d`), no fill
- Min height: 280px desktop / 200px mobile
- Content (centered vertically and horizontally):
  - `InboxOutlined` icon, 48px, color `#8888a8`
  - Heading text: "Drag & Drop your PDF here" — 20px semibold, `#e8e8f0`
  - Body text: "or click to browse" — 14px regular, `#8888a8`, underline on hover
  - Constraint label: "PDF only · Up to 20 MB" — 14px regular, `#8888a8`, always visible (source: D-03)
- Cursor: `pointer` (entire zone is clickable)
- Entire zone is a drop target (not just the inner box)

#### State: drag-over (file hovering above zone)
- Border: 2px solid `#4f6ef7` (accent — border color transitions from dashed to solid)
- Background: `rgba(79, 110, 247, 0.06)` (accent at 6% opacity)
- Icon: changes to `CloudUploadOutlined`, color transitions to `#4f6ef7`
- Heading text changes to: "Release to upload"
- Transition: `border-color 150ms ease, background 150ms ease`

#### State: uploading (file accepted, upload in progress)
- Border: 2px solid `#4f6ef7`
- Background: `rgba(79, 110, 247, 0.04)`
- Content replaces drop zone content:
  - File name (truncated with ellipsis at 280px) — 14px semibold
  - Ant Design `Progress` component, `type="line"`, `strokeColor: '#4f6ef7'`, shows real XHR percentage (0–100%)
  - Status text: "Uploading... {N}%" — 14px regular, `#8888a8`
- Drop zone remains active — new file can be dropped to cancel and replace (source: D-05)

#### State: upload-complete (file successfully in Vercel Blob)
- Drop zone returns to idle visual state (dashed border, original content)
- File info card appears BELOW the drop zone (not replacing it) (source: D-04, D-05)
- Drop zone remains fully interactive for replacement

#### State: error (validation failed)
- Border: 2px solid `#ff4d4f` (error color), `border-radius: 8px`
- Background: `rgba(255, 77, 79, 0.06)`
- Content:
  - `CloseCircleOutlined` icon, 48px, color `#ff4d4f`
  - Error heading: see Copywriting Contract — 14px semibold, `#ff4d4f`
  - Error body: see Copywriting Contract — 14px regular, `#8888a8`
- Error clears on: new drag enter event OR click anywhere in the zone (source: D-07)
- Transition: `border-color 150ms ease, background 150ms ease`

## Interaction Design

### Drag & Drop Flow

```
User drags file over browser window
  → dragenter fires on document
  → Drop zone enters drag-over state (border + bg transition)
  → User releases file
    → dragdrop fires
    → Client validation runs (synchronous, before any upload)
      → Invalid (not PDF OR > 20 MB)
        → Drop zone enters error state
        → No upload initiated
      → Valid
        → Drop zone enters uploading state
        → XHR upload to Vercel Blob begins
        → Progress events update Progress component (0–100%)
        → On complete: drop zone returns to idle, File info card appears
        → On XHR error: drop zone enters error state with network error copy
```

### Click-to-Browse Flow

```
User clicks anywhere in the drop zone
  → (If in error state: clear error, return to idle first)
  → Hidden <input type="file" accept=".pdf"> programmatically clicked
  → File picker dialog opens
  → User selects file
    → Same validation + upload flow as drag & drop
```

### State Machine (useReducer)

```
idle
  ├─ DRAG_ENTER    → drag-over
  ├─ FILE_SELECTED → uploading
  └─ (default)

drag-over
  ├─ DRAG_LEAVE    → idle
  ├─ DROP_VALID    → uploading
  └─ DROP_INVALID  → error

uploading
  ├─ PROGRESS      → uploading (updates percentage)
  ├─ UPLOAD_DONE   → upload-complete
  ├─ UPLOAD_ERROR  → error
  └─ NEW_FILE_DROP → uploading (cancels previous XHR, starts new)

upload-complete
  ├─ NEW_FILE_DROP → uploading
  ├─ RESET         → idle (triggered by "Upload another file")
  └─ (File info card visible, drop zone idle)

error
  ├─ DRAG_ENTER    → drag-over (error clears on new drag)
  ├─ ZONE_CLICK    → idle (error clears on click)
  └─ FILE_SELECTED → uploading
```

### Keyboard Navigation

- `Tab` brings focus to the drop zone container (visible focus ring: 2px solid `#4f6ef7`, `outline-offset: 2px`)
- `Enter` or `Space` on focused drop zone triggers the file picker dialog
- `Tab` from drop zone moves to File info card elements when visible
- "Upload another file" link: `Tab`-focusable, `Enter` activates reset
- "Compress PDF" button: `Tab`-focusable even when disabled (receives focus but does not activate)

### Animation & Transitions

- Drop zone state transitions: `border-color 150ms ease, background-color 150ms ease`
- File info card entrance: `opacity: 0 → 1` over 200ms, `transform: translateY(8px) → translateY(0)` over 200ms (subtle slide-up)
- Error state entrance: same 200ms fade-in
- No animation on progress bar (it reflects real XHR events — fake easing is anti-pattern per SUMMARY.md)
- `prefers-reduced-motion`: all transitions disabled when media query matches

- On successful upload: focus moves automatically to the File info card container (use `useEffect` + `ref.current.focus()`)
- On error: focus remains on the drop zone (already has focus from click or drag action)
- On "Upload another file" click: focus returns to the drop zone

- Body text `#e8e8f0` on `#0d0d0d`: contrast ratio ~14:1 (WCAG AAA)
- Secondary text `#8888a8` on `#0d0d0d`: contrast ratio ~4.7:1 (WCAG AA)

- Accent `#4f6ef7` on `#0d0d0d` for non-text (border/button): passes WCAG AA non-text (3:1)
- Accent `#4f6ef7` as button text on button background: use white `#ffffff` text on accent button background — contrast ~3.5:1 (meets large text threshold)

## Implementation Notes for Executor

1. **Ant Design ConfigProvider** must wrap `<body>` in `layout.tsx` with `theme: { algorithm: theme.darkAlgorithm, token: { ... } }` — see Color section for token overrides.

2. **Upload.Dragger vs custom div:** Use `Upload.Dragger` as the base Ant Design component but override its internal styles via `className` + CSS Modules. Do not rely on Ant Design Upload's built-in file list rendering — it is hidden; File info card is a fully custom component below.

3. **XHR progress:** Vercel Blob client SDK (`@vercel/blob/client`) exposes an `onUploadProgress` callback. Wire this to `dispatch({ type: 'PROGRESS', percent: N })`.

4. **Error boundary:** Wrap the upload card in a React error boundary — any unhandled exception during upload should fall to the error state, not crash the page.

5. **Drag events on document vs zone:** Attach `dragenter` / `dragleave` on `document` to detect when the user enters the browser window with a file. Only update the drop zone visual state when the pointer is over the zone element. Prevent default on `dragover` document-wide to allow dropping.

6. **Mobile tap to browse:** `<input type="file" accept=".pdf" />` hidden with `display: none`. The entire drop zone div has `onClick={() => inputRef.current.click()}`. This satisfies both keyboard and touch interaction.

7. **"Compress PDF" button (Phase 1 placeholder):** Render with `disabled` prop and `title` attribute set to the tooltip copy. Phase 2 will remove `disabled` and add `onClick` handler.
