# Requirements — PDF Compression App

## v1 Requirements

### Upload (UPLOAD)

- [ ] **UPLOAD-01**: Пользователь может загрузить PDF через Drag & Drop на полноэкранную зону
- [ ] **UPLOAD-02**: Пользователь может загрузить PDF через клик и файловый диалог
- [ ] **UPLOAD-03**: Система валидирует тип файла (PDF only) до начала загрузки
- [ ] **UPLOAD-04**: Система валидирует размер файла (максимум 20 MB) до начала загрузки
- [ ] **UPLOAD-05**: Пользователь видит прогресс загрузки файла в реальном времени (XHR progress events)

### Compression (COMP)

- [ ] **COMP-01**: Пользователь может выбрать один из трёх пресетов сжатия: Maximum Compression / Balanced / High Quality
- [ ] **COMP-02**: Пресет Balanced выбран по умолчанию без действий пользователя
- [ ] **COMP-03**: Система сжимает PDF на сервере (pdf-lib + sharp через Next.js API route)
- [ ] **COMP-04**: Пользователь видит индикатор обработки во время сжатия

### Results (RES)

- [ ] **RES-01**: После сжатия пользователь видит статистику: исходный размер, сжатый размер, процент и МБ экономии
- [ ] **RES-02**: Пользователь может скачать сжатый файл одним нажатием
- [ ] **RES-03**: Скачиваемый файл автоматически получает имя `[оригинал]-compressed.pdf`
- [ ] **RES-04**: После скачивания пользователь видит CTA "Сжать другой файл" для сброса состояния

### Error Handling (ERR)

- [ ] **ERR-01**: Пользователь видит понятное сообщение об ошибке при попытке загрузить не-PDF файл
- [ ] **ERR-02**: Пользователь видит понятное сообщение об ошибке при файле больше 20 MB
- [ ] **ERR-03**: Пользователь видит понятное сообщение об ошибке при сбое сжатия на сервере

### Infrastructure (INFRA)

- [ ] **INFRA-01**: Загрузка файла использует Vercel Blob client-upload (обход лимита 4.5 MB на тело запроса)
- [ ] **INFRA-02**: Загруженные файлы удаляются из Vercel Blob после скачивания или через TTL
- [ ] **INFRA-03**: Приложение развёртывается и работает на Vercel

### UI/UX (UX)

- [ ] **UX-01**: Интерфейс адаптивен для мобильных устройств
- [ ] **UX-02**: Путь от загрузки до скачивания занимает максимум 3 клика
- [ ] **UX-03**: Скачивание файла работает корректно в Safari/iOS (Content-Disposition: attachment)

## Out of Scope

- **Batch processing** — удваивает сложность UI state machine и серверной нагрузки. Деферирован.
- **Dashboard / история файлов** — требует персистентного хранения и аутентификации. Деферирован.
- **Аутентификация** — не нужна в MVP, анонимная загрузка покрывает use case.
- **OCR / AI-оптимизация** — долгосрочная цель, нет в MVP.
- **Desktop app** — долгосрочная цель.
- **Custom DPI input** — технические пользователи хотят, но общая аудитория бросает при виде. Named presets покрывают 95% intent.
- **Email delivery результатов** — добавляет SMTP инфраструктуру и риски спама. Browser download быстрее.
- **PDF editing (merge/split/rotate)** — убивает single-purpose UX clarity.
